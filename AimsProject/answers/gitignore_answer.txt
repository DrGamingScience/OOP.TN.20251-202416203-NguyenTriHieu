Tại sao những đường dẫn trên nên được thêm vào .gitignore (nếu không thêm 
thì sao)?
Việc này là cần thiết do để duy trì một Git lưu trữ mã nguồn, cốt lỗi, không cần push những thư mục đó. 
Bin chứa các file biên dịch, output tự động sinh ra của mã nguồn. Các file còn lại cũng liên quan tới cấu hình,
nên khuyến cáo không nên sử dụng hoặc push pull liên tục.
Nếu như không thêm thì khi các thành viên dùng các bản IDE khác nhau, chỉnh sửa sẽ gây không hợp nhất được code.
Tăng dung lượng Repo không cần thiết
Lỗi build, các file classpath sẽ tạo tham chiếu không đúng trên máy người khác nếu họ clone về sử dụng, vì 
những file cấu hình đó phụ thuộc vào máy người dùng


Nếu sử dụng những công cụ khác để lập trình Java (như IntelliJ IDEA hoặc VSCode), 
những đường dẫn nào sẽ được thêm vào .gitignore?

Intellij IDEA: .idea/ workspace.xml, misc.xml, vcs.xml em không chắc chắn vì không sử dụng bao giờ
VS Code: .vscode/, lib/

Trong trường hợp bạn đã lỡ 
commit các file tạm lên Git, nhưng sau đó bạn đã thêm chúng vào .gitignore và muốn Git 
ngừng theo dõi chúng mà không xóa khỏi máy, bạn sẽ cần thực hiện những thao tác nào?

Đầu tiên phải git rm -r --cached + đường dẫn
Sau đó git commit -m lại , ghi thông điệp để biết hành động vừa làm gì
Sau đó thì push lại lên github là được
